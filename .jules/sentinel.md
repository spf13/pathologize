## 2024-06-02 - Path Sanitization Order of Operations Bypass
**Vulnerability:** The path sanitization routine applied length truncation after validating against Windows reserved names (like CON) and stripping trailing characters. An attacker could bypass the filter by supplying a payload like "CON" + (252 spaces) + "x". The filter would not match "CON", but the truncation step would remove the "x", leaving a string with trailing spaces that Windows would evaluate as the reserved device "CON".
**Learning:** Path sanitization must always perform length bounding/truncation before evaluating semantic meaning, checking for reserved names, or trimming trailing spaces/dots. Truncating after these validation steps can create new string suffixes that bypass the filter logic.
**Prevention:** Ensure that truncation is one of the very first operations performed during path sanitization, immediately following character filtering, and strictly prior to validation checks or trimming.
