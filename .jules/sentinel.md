## 2025-04-03 - Windows Reserved Name Path Validation Bypass via Multiple Extensions
**Vulnerability:** The path validation logic to prevent the use of Windows reserved names (e.g., `CON`, `PRN`, `AUX`) was bypassed if the filename contained multiple extensions (e.g., `CON.tar.gz` or `CON.txt.txt`). The `filepath.Ext(filename)` function only extracts the *final* extension (e.g., `.gz` or `.txt`), leaving the intermediate extensions attached to the base name (e.g., `CON.tar` or `CON.txt`). This bypassed the exact string match validation for reserved names.
**Learning:** When performing path sanitization, especially for Windows reserved names, the true base name of the file must be isolated by stripping *all* extensions (everything after the first dot), because Windows treats any extension on a reserved device name as the device itself.
**Prevention:** Instead of using `filepath.Ext()`, split the filename on the first dot (`strings.SplitN(filename, ".", 2)[0]`) to reliably isolate the true base name for validation. When reconstructing the filename, use `strings.TrimPrefix(filename, basefilename)` to preserve all original extensions.
