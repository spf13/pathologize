## 2024-05-31 - Path Sanitization Order of Operations Vulnerability
**Vulnerability:** The `Clean` function truncated filenames *after* checking for Windows reserved names (e.g., `CON`, `PRN`) and stripping trailing characters. An attacker could craft a filename like `CON` + 252 spaces + `a` (256 chars). The trailing `a` bypasses trailing character removal and reserved name checks. The final truncation then cuts the `a`, leaving `CON` + 252 spaces (255 chars), which Windows silently evaluates as the reserved `CON` device, allowing potential device hijacking or denial of service on Windows hosts.
**Learning:** Path sanitization must evaluate semantic properties (like reserved names or trailing bypass chars) *after* length bounding. Truncating at the end of a pipeline can create new, unchecked malicious suffixes or exact matches.
**Prevention:** Always perform length bounds checking/truncation before evaluating semantic meaning, checking for reserved names, or trimming trailing spaces/dots.
